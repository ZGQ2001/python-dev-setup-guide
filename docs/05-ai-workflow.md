# 2.1 AI 辅助编程：工具选择与高效协作

>💡 本章目标：了解市面上有哪些 AI 编程工具、各自适合什么场景、免费能用多少，掌握让 AI 写出可靠代码的沟通方法和验收流程。

你现在打开 VS Code，Copilot 插件已经在右上角亮着了。打开浏览器，豆包和 ChatGPT 随时能用。但**这些工具不是"谁更好"的问题——它们解决的是完全不同的需求**。选错工具，就像用螺丝刀敲钉子：也能敲，但累死你。

本章先帮你理清四类 AI 编程工具的定位和免费额度，再教你如何跟 AI 沟通让它输出你真正需要的代码，最后给出验收标准和踩坑清单。

---

## 一、四种 AI 编程模式：你的工具箱

把 AI 编程工具想象成四种不同的"帮手"，各自站在不同的位置：

```
你（写代码的人）
  │
  ├── [编辑器内补全] Copilot / Cursor / Windsurf
  │    站在你肩膀后面，看你打字，随时递工具
  │
  ├── [网页对话式] ChatGPT / 豆包 / DeepSeek / Gemini
  │    坐在隔壁座位上，你问它答，它写一整段给你
  │
  ├── [终端智能体] Claude Code / Codex CLI
  │    自己跑进你的项目文件夹，读文件、改代码、跑测试
  │
  └── [本地部署] Ollama + Continue / Aider
       完全在你自己的电脑里运行，不联网，零费用
```

### 1. 编辑器内补全：影子助手

你在 VS Code 里写代码，它**在光标后浮现灰色字**，按 `Tab` 接受，按 `Esc` 忽略。

**代表工具：**
- **GitHub Copilot**（VS Code / JetBrains 插件）
- **Cursor**（独立编辑器，内置 AI）
- **Windsurf**（独立编辑器，Codeium 驱动）
- **Trae**（字节跳动免费 IDE，内置 Claude + GPT）

**适合的场景：**
- 写重复性代码（循环、条件判断、数据转换）
- 填函数体（你已经写好函数名和参数，它补实现）
- 写注释后自动生成代码

**不适合的场景：**
- 从零设计一个模块（太大，补全不了）
- 重构（要看多个文件之间的关系）
- 学习一个新库（你需要解释，不是补全）

### 2. 网页对话式：隔壁的老工程师

打开浏览器，在对话框里用自然语言描述需求，AI 给你返回一段完整代码。你**复制 → 粘贴到编辑器 → 调试验收**。

**代表工具：**

| 工具 | 网址 | 免费额度 | 特点 |
|------|------|---------|------|
| **ChatGPT** | chatgpt.com | GPT-5 mini 免费，GPT-5 有限额 | 综合能力强，英文语境最佳 |
| **豆包** | doubao.com/chat/coding | **完全免费** | 中文原生，网页生成出色，适合零基础 |
| **DeepSeek** | chat.deepseek.com | 免费 | 推理能力强，代码逻辑好，中文友好 |
| **Gemini** | gemini.google.com | 免费额度大 | Google 生态集成好，多模态 |
| **Kimi** | kimi.moonshot.cn | 免费 | 长文本处理强，中文理解好 |

**适合的场景：**
- 从零生成一个完整功能模块
- 解释看不懂的代码或报错信息
- 学习新技术（"给我一个 pandas 处理 CSV 的最小示例"）
- 重构一段代码（贴进去 → "改成更清晰的写法"）
- 生成网页或可视化界面（豆包擅长）

**不适合的场景：**
- 边写边补（来回切窗口太慢）
- 需要读取项目上下文（除非你手动贴文件内容）

### 3. 终端智能体：自己动手的机器人

你给一个任务描述，它自己进你的项目文件夹——读文件、写代码、跑测试、改 bug——完成后再回来报告。

**代表工具：**
- **Claude Code**（Anthropic，终端运行，需付费）
- **Codex CLI**（OpenAI，终端运行）
- **Gemini CLI**（Google，终端运行）
- **Aider**（开源，可本地模型）

**适合的场景：**
- 多文件改动（改一个功能涉及 5 个文件）
- 自动修复测试失败
- 批量重命名 / 重构
- "帮我给整个项目加类型注解"

**不适合的场景：**
- 新手——它改动大、速度快，出问题你未必能拦住
- 免费预算——Claude Code 无免费额度，API 按量计费

### 4. 本地部署：不联网的私人助手

在你的电脑上安装大模型，完全离线运行。不需要网络，不需要 API key，数据不离开你的硬盘。

**代表方案：**
- **Ollama**（模型运行框架） + **Continue**（VS Code 插件）= 本地 Copilot
- **Ollama + Aider** = 本地终端智能体

**需要的硬件：**
- 至少 16GB 内存
- 推荐有独立显卡（显存 6GB+），纯 CPU 也能跑但慢
- 模型文件 2~8GB 不等，需要硬盘空间

**适合的场景：**
- 处理敏感数据（客户信息、公司内部代码）不能上传云端
- 网络受限的环境
- 长期使用，不想付月费

**不适合的场景：**
- 笔记本电脑配置不够
- 需要最强的代码生成能力（本地模型比云端弱）

> ⚠️ **新手建议**：先不用碰本地部署。用网页版 AI 或 Copilot Free 跑通整个工作流，等你确定"AI 编程确实能提高我的效率"，再考虑本地部署。

## 二、免费额度速查：能免费用多少

### Copilot Free（VS Code 插件）

GitHub 提供的免费套餐，不需要信用卡，注册 GitHub 账号就能用。

| 功能 | 免费额度 | 够不够用 |
|------|---------|---------|
| **代码补全**（灰色字） | 每月 2,000 次 | ✅ 新手够用。日均 66 次，不是重度编程绰绰有余 |
| **对话提问**（Copilot Chat） | 每月 50 次 | ⚠️ 偏紧。平均每天 1-2 次对话 |
| **Agent 模式**（自动改代码） | 每月 50 次 | ⚠️ 同上 |
| 可用模型 | Haiku 4.5, GPT-5 mini | ✅ 基础模型，写简单代码够用 |

> 💡 **省额度技巧**：Copilot Chat 的 50 次不是按"发送消息"算的，是按"一次完整对话"算的。在一个对话窗口里追问多次只算一次。所以不要每问一个问题就开新窗口——在一个窗口里把事聊透。

**额度用完怎么办？** Copilot 的补全和对话会暂停，**下个月自动重置**。不会扣费。

### 完全免费的工具（不花钱，不受限）

| 工具 | 使用方式 | 限制 | 推荐理由 |
|------|---------|------|---------|
| **豆包 AI 编程** | 网页 | 无次数限制 | 零门槛，中文原生，网页生成强。适合完全不会写代码的人 |
| **DeepSeek** | 网页 | 无次数限制 | 推理能力突出，代码逻辑好 |
| **Trae** | 下载 IDE | **完全免费**（字节补贴） | 内置 Claude + GPT，中文原生，相当于免费版 Cursor |
| **Gemini** | 网页 | 每日限额高，普通用户用不完 | Google 搜索 + AI 联动 |
| **Continue + Ollama** | VS Code 插件 + 本地模型 | 取决于你的电脑性能 | 完全离线，零费用，数据不出本地 |

### 付费但值得参考的

| 工具 | 价格 | 值在哪 |
|------|------|--------|
| Copilot Pro | $10/月（约 ¥72） | 补全无限 + 对话 300 次/月 + 代码审查 |
| Cursor Pro | $20/月（约 ¥145） | 体验最好的 AI IDE，自动理解整个项目 |

## 三、如何让 AI 准确理解你的需求

AI 没有读心术。你对项目背景的了解、数据的格式、你想达到的效果——这些全在你脑子里，AI 一概不知。**你给出的信息越精确，AI 返回的代码越接近你的预期。**

### 1. 描述"要什么结果"，而不是"怎么写代码"

AI 不需要你告诉它用哪个库的哪个函数——你只需要描述输入和期望的输出。

**对比：**

```
❌ "写一个 pandas 的 groupby 然后 agg 取平均值"
    → 你已经在脑子里替 AI 做了技术选型。但 pandas 是最优解吗？
    如果数据只有 100 行，可能纯 Python 更简单。

✅ "我有一个表格文件，第一列是部门名称，第二列是员工工资。
    帮我统计每个部门的平均工资，输出到新的表格文件。"
    → 描述业务目标，让 AI 选择实现方式。
```

### 2. 分步给指令，不要一次全扔

**人类对新人的本能是"一口气全交代清楚"**——但在 AI 世界里，这会让它顾此失彼。

```
❌ 一次性扔：
  "帮我写一个程序，读取文件夹里所有的 Excel 文件，
   合并成一张总表，去重，按日期排序，去掉负数，
   按部门分组统计，画柱状图，输出 PDF 报告。"

✅ 分步：
  第一步："写一个函数，读取文件夹里所有 .xlsx 文件，返回文件名列表"
  → 跑通 → 确认文件读取正确

  第二步："把这些文件合并成一张表，遇到完全相同的行只保留一条"
  → 跑通 → 目测行数对不对

  第三步："去掉数值列里的负数，替换为空"
  → 跑通

  ...以此类推
```

**为什么分步更高效？** 如果一步到位出了 bug，你面对的是 200 行谁写的、哪一步出错的代码。分步跑，任一步出错，你只需要排查那一步的输入输出。

### 3. 提供上下文（让 AI 知道你的环境）

每次新对话开始，告诉 AI 你的基本环境。不需要很复杂——三句话够：

```
我正在用 Python 3.12，包管理用 uv。
项目中已经装了 openpyxl 和 pandas。
代码统一放在 src/ 目录下，数据文件放在 data/ 目录下。
```

> 💡 把这 3-4 句做成一个**文本片段存起来**，每次开新对话直接粘贴，省得反复打字。

### 4. 给例子（输入 → 期望输出）

AI 最擅长的是**模仿**。如果你能给出一个输入和期望输出的对应关系，AI 的准确率会高很多。

```
✅ "帮我写一个函数 format_date。输入示例：'2024-01-05'
   期望输出：'2024年1月5日'。如果输入是空的，返回空字符串。"

✅ "帮我写一个函数 classify_score。规则是：
   90-100 → '优秀'，70-89 → '良好'，60-69 → '及格'，60以下 → '不及格'。
   输入 85 应该返回 '良好'。"
```

## 四、验收 AI 代码：三关必过

AI 生成的代码在通过下面三关之前，**只是草稿**。直接拿去用 = 闭着眼踩油门。

### 第一关：读懂代码（你理解每一行在干什么）

**为什么是第一关？** 如果你不理解代码逻辑，出了 bug 你无从排查。

| 排查点 | 看什么 | 为什么 |
|--------|--------|-------|
| 陌生库 | `from cryptography import ...` | AI 会引入不存在、不需要的库。确认这个库真实存在 |
| 硬编码 | 文件路径、账号密码直接写在代码里 | 换电脑就挂，必须改用配置文件 |
| 假 API | `df.to_civ_format()` | AI 会编造不存在的函数名。确认每个函数调用是真实存在的 |
| 魔法数字 | `if value > 150:` | 150 是什么意思？应该改成有名字的常量 |

**实操方法：** 看不懂的代码行，复制到 AI 对话框追问"这一行 `xxx` 在做什么?"直到你能向别人讲清楚。

### 第二关：红线全清（VS Code 零报错）

打开 VS Code 底部的状态栏，确认两项：

```
Pylance（类型检查）：
  ❌ 有数字 → 不许往下走
  ✅ 显示 0 → 过关

问题面板（Ctrl+Shift+M）：
  ❌ 有红色 ×  → 点开看，逐条修
  ✅ 空白 → 过关
```

如果你按第一章配了 `settings.json`，按 `Ctrl+S` 保存时 Ruff 会自动修一部分格式问题。

### 第三关：实际跑通（不只是不报错）

```bash
# 在终端执行
uv run python src/你的文件.py
```

**验收标准：**

| 测试方式 | 具体做法 |
|---------|---------|
| 正常输入 | 用一个简单、数据量小的真实文件跑一遍，目测输出结果对不对 |
| 异常输入 | 故意给空文件、错误格式的文件，看报错信息是不是清晰 |
| 边界输入 | 给一条数据、给大量数据，看会不会崩溃或卡死 |

> ⚠️ **AI 代码最常见的假把式**：不报错但输出为空，或者悄悄跳过了某些数据行。**必须用真实数据目测输出内容**——不能只看"终端没报错"。

## 五、AI 输出不符合预期怎么办

AI 第一次就写出完美代码的概率很低。但你不应该"算了，自己写吧"——你应该**追问**，像对实习生提修改意见一样。

### 第一步：诊断问题类型

| 问题表现 | 可能原因 | 解决方向 |
|---------|---------|---------|
| 代码跑不起来，报错 | AI 用了不存在的 API | 贴报错信息追问 |
| 代码能跑但结果不对 | AI 理解错了你的需求 | 重新描述期望，给输入输出示例 |
| 代码风格不一致 | 不同轮次 AI 用了不同写法 | 在 AGENTS.md 里定规范，见第六节 |
| 代码太复杂 | AI 倾向于"过度工程" | 明确要求"用最简单的方式实现，不要引入多余的抽象" |
| 引入了不需要的库 | AI 默认用自己熟悉的库 | 明确"只用标准库"或列出允许的库 |

### 第二步：追问模板

```
✅ "这个版本能跑了，但我发现 [具体问题]。帮我改一下：
   - [具体要求 1]
   - [具体要求 2]
   只改这一处，不要动其他部分的代码。"

✅ "这段代码逻辑对了，但太啰嗦。能用更简洁的方式重写吗？
   保持输入输出不变，行数控制在 30 行以内。"

✅ "这样写能跑，但不是我想要的效果。
   我给你一个例子：输入 [A] 应该得到 [B]。
   你写的版本输入 [A] 得到了 [C]，问题出在哪？"
```

### 第三步：换工具

同一个任务，不同的 AI 给出的结果可能天差地别。

```
Copilot 给的代码太复杂 → 拿同样的需求去问 DeepSeek
ChatGPT 没理解中文需求 → 换豆包试试
豆包生成的交互效果不对 → 换 Gemini 看看
```

> 💡 **铁律：如果一个 AI 追问三轮还是不 work，换一个。** 死磕同一个 AI 是最常见的效率黑洞。不同 AI 的底层模型不同，"擅长领域"也不同。

## 六、让 AI 长期写出好代码：项目规范文件

你每开一次新对话，AI 对你的项目一无所知。如果你每次都要重新说一遍"我们用 Python 3.12，包管理用 uv，代码放 src/ 里"，那不是在编程——是在做客服。

### 解决方案：在项目根目录创建一个 AGENTS.md

把这个文件放在项目根目录（和 `pyproject.toml` 同级）。每次新对话开始时，把内容贴给 AI——或者某些工具（Cursor 等）会自动读取。

```markdown
# AGENTS.md — 项目约定

## 环境
- Python 3.12，包管理用 uv（永远用 `uv run` 而不是 `python`）
- 依赖清单在 pyproject.toml，不要引入新库除非明确允许

## 目录结构
- src/ — 所有业务代码
- data/ — 数据文件
- output/ — 输出结果

## 代码规范
- 所有函数必须有类型注解
- 变量名用小写下划线（snake_case）
- 文件路径用正斜杠 `/`，相对路径以项目根为起点 `./data/xxx`
- 中文注释

## 禁止
- 禁止硬编码路径（绝对路径）
- 禁止在代码里写密码或密钥
- 禁止引入项目里没在用的库
```

> 💡 如果你觉得维护 AGENTS.md 麻烦，**至少在新对话的第一句话里说明 3 件事**：Python 版本、包管理工具、目录结构。

### GitHub 参考

很多开源项目已经把 AGENTS.md 或 CLAUDE.md 放在仓库根目录。你可以直接参考：

- **[Addy Osmani 的 AI 编程工作流](https://addyosmani.com/blog/ai-coding-workflow/)** — Google 工程总监分享的 AI 辅助编程方法论，核心原则是"先出规格再写代码"
- **[GitHub Copilot 官方最佳实践](https://docs.github.com/en/copilot/get-started/best-practices)** — 微软官方指南
- **[Claude Code 官方最佳实践](https://code.claude.com/docs/en/best-practices)** — Anthropic 的推荐用法，很多原则对 AI 编程通用

## 七、常见误区与避坑指南

| 误区 | 为什么错 | 正确做法 |
|------|---------|---------|
| "AI 写的代码不报错就是对的" | 逻辑正确 ≠ 业务正确。输出为空、漏数据、单位搞错——这些都不会报错 | 用真实数据验证输出内容 |
| "贴越长越详细的提示词 AI 越懂" | 超过 500 字的提示词，AI 会选择性忽略靠后的内容 | 拆成多个小任务，一个需求一次对话 |
| "Copilot 灰色建议直接 Tab" | Copilot 会猜错变量名、用不存在的 API、写出它自己也没验证过的逻辑 | 先扫一眼灰色字，逻辑通再 Tab |
| "报错直接贴给 AI 让它修" | AI 倾向于"修症状"而不是"修根因"——把 `try/except` 包住报错行就完事了 | 先自己看报错堆栈，找到**导致报错的根源代码行**，再让 AI 改那一行 |
| "代码跑通了就不需要类型注解" | 一个月后回来看，没类型注解跟读天书一样 | 写函数的同时写好类型注解，不"回头补" |
| "AI 推荐的库直接 pip install" | AI 会推荐过时版本、不兼容版本、甚至**不存在的库**（AI 幻觉） | 先搜 [pypi.org](https://pypi.org) 确认这个库存在、最新版本是什么 |
| "没用完免费的，没必要付费" | 免费模型的代码质量比付费模型差一个级别。如果你每天有 2 小时以上在写代码 | "先用免费，效率被额度卡住的那天再付费" |
| "不同 AI 都差不多" | 同一需求给 Copilot、ChatGPT、豆包、DeepSeek，结果可能完全不同 | 一个 AI 追问三轮不行就换。不做死磕型选手 |
