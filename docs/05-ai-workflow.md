# 2.1 AI 辅助编程：工具选择与高效协作

>💡 本章目标：了解有哪些免费 AI 工具能帮你写代码，学会用自然语言描述需求让 AI 生成代码，掌握"复制→粘贴→运行→报错就回去问"的工作闭环。

你不会写代码？没关系。AI 会写。你只需要做两件事：**用中文把需求说清楚**，然后**把代码复制到 VS Code 里跑一下**。

本章不要求你懂任何编程基础。所有操作都只需要浏览器和 VS Code——这两个你第一章已经装好了。

## 一、你能用的免费 AI 工具

打开浏览器就能用。不需要安装，不需要配置，不需要信用卡。

### 主力推荐（选一个，固定用熟）

| 工具 | 网址 | 费用 | 为什么推荐 |
|------|------|------|-----------|
| **豆包** | doubao.com/chat/coding | **永久免费** | 中文原生，不需要翻译思考，网页生成最强 |
| **DeepSeek** | chat.deepseek.com | **永久免费** | 代码逻辑最严谨，报错修复能力强 |
| **Kimi** | kimi.moonshot.cn | **永久免费** | 长文本处理强，能一次读完很长的需求描述 |

> 💡 **新手推荐豆包 + DeepSeek 组合**：日常生成代码用豆包（中文最自然），报错修不好时用 DeepSeek（逻辑最强）。两个都用免费，不需要二选一。

### 备选（有免费额度但有限）

| 工具 | 网址 | 免费额度 | 适合什么时候用 |
|------|------|---------|---------------|
| **ChatGPT** | chatgpt.com | GPT-5 mini 免费 | 英文需求、国际通用场景 |
| **Gemini** | gemini.google.com | 每日额度高 | 截图问代码（拍一张表格，让它生成读取代码） |

**Copilot** 和 **Cursor** 是给会写代码的人用的——它们在编辑器里自动补全代码，需要你本身在打字。你不会敲代码，这些工具对你没用。**用网页版就够了。**

## 二、选工具对照表（按"我想做什么"选）

| 我想做什么 | 用哪个 | 怎么做 |
|-----------|--------|--------|
| 从零生成一个新功能 | 豆包 / DeepSeek | "帮我写一个 Python 程序，读取 Excel 文件里的……" |
| 报错了不知道怎么修 | DeepSeek | 把红色报错信息全部复制 → 贴到对话框 → "这个报错怎么修？" |
| 看不懂一段代码 | 豆包 / DeepSeek | 把代码复制 → 贴到对话框 → "逐行解释这段代码在做什么" |
| 想做一个网页展示数据 | 豆包 | 描述你想展示什么 → 豆包生成 HTML 文件 → 双击打开 |

> 💡 **铁律：一个 AI 追问三轮还不 work，换另一个。** 同一个任务，换豆包和 DeepSeek 经常一个翻车另一个跑通。

## 三、如何让 AI 写出你想要的代码

AI 不会读心术。你对数据格式的了解、业务的规则、你想达到的效果——全在你脑子里，AI 一概不知。**给的信息越精确，出来的代码越靠谱。**

### 1. 描述"要什么结果"，不是"怎么写"

```
❌ "用 pandas 写个 groupby 然后 agg 取平均值"
    → 你已经在替 AI 做技术决策。但你自己不写代码，这些术语你也不懂。

✅ "我有一个表格，第一列是部门名称，第二列是员工工资。
   帮我统计每个部门的平均工资，输出一个新的表格文件。"
    → 描述业务需求。AI 自己决定用什么方法实现。
```

### 2. 告诉我你的文件长什么样

AI 看不到你的电脑。如果你不说文件格式，它只能瞎猜。

```
❌ "帮我读一下那个表"
    → AI 不知道文件名、不知道格式、不知道有哪些列。

✅ "我的 Excel 文件叫 2025年1月检测数据.xlsx，放在项目文件夹的 data/ 目录里。
   表头有 5 列：构件编号、检测类型、检测人、检测日期、是否合格。
   帮我写代码读取这个文件，统计每种检测类型的合格率。"
    → AI 知道文件名、知道目录、知道列名、知道要算什么。能直接生成可用代码。
```

### 3. 给个例子——效果翻倍

AI 最擅长**模仿**。如果能给一个"输入什么 → 期望输出什么"的例子，准确率会大大提高。

```
✅ "帮我写一个函数，把日期格式转换一下。
   输入：'2024-01-05'
   输出：'2024年1月5日'
   如果输入是空的，返回空就行。"
```

### 4. 分步问，不要一次全扔

任务越复杂，一次扔给 AI 越容易出错。拆成小步。

```
❌ 一次扔：
  "帮我写程序：读取所有Excel→合并→去重→排序→过滤→分组统计→画图→输出PDF"

✅ 先问：
  "帮我写代码：读取 data/ 目录下所有 .xlsx 文件，列出所有文件名"
  → 跑通，确认文件找对了

  再问：
  "在上一步的基础上，把这些文件合并成一张总表，
   完全相同的行只保留一条。"
  → 跑通，确认合并结果

  再问：
  "在上一步的基础上，按日期列从早到晚排序，去掉数值为负的行。"
  → 以此类推
```

## 四、拿到代码后：四步验收

AI 给的代码不是成品——你得验证它真的能用。

### 第一步：复制粘贴到 VS Code

1. AI 对话框中选中代码 → `Ctrl+C` 复制
2. 打开 VS Code → 在左侧文件列表找到 `src/你的项目名/` → 右键 → 新建文件 → 输入文件名（如 `统计脚本.py`）
3. 在新建的文件里 `Ctrl+V` 粘贴
4. `Ctrl+S` 保存

### 第二步：跑起来

在 VS Code 底部点 `终端`，输入：

```bash
uv run python src/你的项目名/你刚保存的文件.py
```

按回车。看输出结果。

### 第三步：验收——用眼睛看

| 检查什么 | 怎么检查 |
|---------|---------|
| 输出内容对不对 | 打开生成的表格/图片，用眼睛看。数字合理吗？有没有漏掉的？ |
| 有没有红色报错 | 终端里出现大段红色文字 = 报错了，看第五节 |
| 有没有卡住不动 | 超过 30 秒没反应 = 可能卡死了，按 `Ctrl+C` 停止 |
| 原来的文件变没变 | 去 data/ 目录看一眼，确保原始数据没被覆盖 |

### 第四步：报错了怎么办

**把报错信息完整复制（用鼠标选中红色文字 → `Shift+Insert` 或右键粘贴到 AI 对话框）。

```
你："代码报错了，下面是报错信息。帮我修一下，不要改别的部分。"

[粘贴报错信息]

→ AI 会给你修复后的代码
→ 重复：复制 → 粘贴到 VS Code → 保存 → 再跑一次
```

## 五、AI 写的不对怎么办

### 第一步：追问，不要放弃

AI 第一版代码经常不完美。像跟同事提修改意见一样，不满意就继续说。

```
✅ "能跑了，但输出的表格里部门名称变成了乱码。帮我修一下编码问题。"

✅ "能跑了，但速度太慢。有没有办法让它跑得更快？不用解释原理，直接给改好的代码。"

✅ "这样是能跑，但不是我要的效果。我希望的结果是 [描述期望结果]。
   现在得到的是 [描述当前结果]。差在哪里？"
```

### 第二步：追问三轮仍不行 → 换工具

同一个 AI 追了三轮还是不对，把同样的需求贴给另一个 AI。

```
豆包不 work → 去问 DeepSeek
DeepSeek 不 work → 去问 Kimi
Kimi 也不 work → 回来再问豆包，但换一种描述方式
```

## 六、让它一直写出好代码：项目规范文件（做一次，管一辈子）

AI 每次新对话都对你的项目一无所知。创建下面这个文件后，每次新对话把它贴给 AI——你不需要背，复制粘贴就行。

在你的项目根目录（`D:\CodeProjects\你的项目名\`）新建一个文件叫 `AGENTS.md`：

> 💡 你不需要自己写这个文件。把下面这段话发给 AI，让它帮你生成，然后保存到项目根目录：

```
帮我创建 AGENTS.md 文件，包含以下内容：
- Python 3.12 + uv 包管理器
- 我的项目名是 [填写你的项目名]
- 代码放在 src/[项目名]/ 里
- 数据文件放在 data/ 里
- 输出文件放在 output/ 里
```

**以后每次开新 AI 对话，第一句话先发三件事：**

```
"我正在做 Python 项目，Python 3.12，用 uv 管理，代码在 src/ 里。下面是 AGENTS.md 文件：[粘贴内容]"
```

## 七、常见误区

| 误区 | 实际情况 |
|------|---------|
| "AI 写的代码不报错就是对的" | 不报错 ≠ 结果正确。可能输出为空、漏掉了某些行。**必须用眼睛看输出内容** |
| "提示词越长越好" | 太长的需求，AI 会忽略后面的。**一次只问一个问题** |
| "报错了 = AI 不行，自己想办法" | 报错修好是 AI 的强项。**把报错贴回去就行** |
| "豆包和 DeepSeek 差不多" | 同一个需求，一个可能完美、一个可能翻车。**三轮不修就换** |
| "AI 推荐的库直接 pip install" | AI 会推荐不存在的库。**让 VS Code 终端告诉你装没装上，没装上把报错贴回 AI** |
| "我不用学任何编程知识" | 你需要知道：什么是文件路径、什么是终端、怎么复制粘贴。第一章已经讲完了 |
| "AI 能帮我写 = 不需要理解需求" | 没人比你更懂你的业务。数据里哪些是正常值、哪些是异常值——只有你知道 |
